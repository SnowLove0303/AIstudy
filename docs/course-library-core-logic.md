# 课程库核心逻辑

## 目录与父子级

- 课程库以主思维导图的 `nodeData` 作为唯一目录树，目录只呈现这棵树的父子级关系。
- 章节、节、点都可以作为可点击目录项，非最末子级也必须允许承载知识点正文。
- 折叠只影响左侧目录可见性，不能修改主思维导图，也不能强制回弹覆盖用户手动折叠。

## 知识点与知识笔记

- 知识点正文按 `nodeId` 保存到 `knowledgePoints`，知识笔记按同一个 `nodeId` 保存到 `knowledgeDocuments`。
- 父级节点没有手写正文时，可以临时显示下级提纲，但临时提纲不能自动写成正式知识点。
- 用户一旦编辑父级知识点，该内容就是该父级节点自己的正文，不能被子级内容替换。

## 思维导图与同步

- 主思维导图负责全局结构，分支导图只以当前节点为根展示和编辑局部结构。
- 分支导图保存时必须写回对应 `nodeId` 的分支，不能写到父级、根节点或其他分支。
- 同步开启时，分支修改要同步回主图和相关父级分支；同步关闭时，只保留当前分支视图改动边界。

## 持久化边界

- 课程数据必须同时保持 JSON 与 MySQL 摘要一致，重启后目录、折叠、知识点、知识笔记和分支导图都应恢复。
- 目录快照只能冻结编号显示，不能改变真实父级、深度、节点归属或已有节点标题。
- 系统自动生成的父级提纲必须带机器标记，清理逻辑不能因为正文包含“思维导图”等词就删除用户内容。

## 原始基准

思维导图：构建目录分支与全局父子级

目录：呈现各父子级标题，这一点已经有相关逻辑。如：总-第一章-第一节-第一点。

知识点可以对各个父子级进行详细内容分级，不影响思维导图。

知识点可以切换到该知识点对应的子分支思维导图进行编辑，该编辑可以同步到总思维导图和该分支的所有父级思维导图，也可以通过关闭同步功能防止修改父子级分支。
