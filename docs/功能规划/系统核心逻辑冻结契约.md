# 系统核心逻辑冻结契约

## 目标

从本版本开始，系统只允许围绕「课程 + 思维导图 + Word 文档」这条主链路扩展。后续新增 AI、导出、复习卡、素材库、端口管理等功能时，不能改变这条链路的数据归属和身份规则。

## 唯一身份链

1. `course_id` 是课程容器身份。
2. `mind_map_id` 是课程下唯一导图身份。
3. `node_id` 是导图节点稳定身份，来自 `simple-mind-map` 节点 `data.uid`。
4. Word 详情文档只允许通过 `course_id + mind_map_id + node_id` 绑定。
5. 标题、目录路径、UI 选中状态、节点显示顺序都不能作为跨功能绑定键。

## 思维导图契约

- 导图编辑器固定为 `simple-mind-map`。
- 导图快照协议固定为 `schemaVersion: 1` 和 `editor: "simple-mind-map"`。
- 节点树 `root.children` 是目录父子级的唯一来源。
- 节点标题允许重复，重命名不得改变节点 ID。
- 目录只允许由 `buildMindMapOutline(root)` 生成。
- 目录折叠只属于 UI 状态，不落库，不反写导图树。
- 删除节点后，节点投影软删除，同时相关 Word 文档先软删除。

## Word 文档契约

- Word 编辑器固定为 `@hufe921/canvas-editor`。
- Word 快照协议固定为 `schemaVersion: 1` 和 `editor: "aistudy-word"`。
- `knowledge_documents` 只保存当前文档指针和轻量元数据。
- `knowledge_document_snapshots` 保存实际编辑器内容快照。
- 打开课程时不得加载整门课所有 Word 内容。
- 切换节点或切换编辑器模式前必须先保存当前未落库内容。
- 保存 Word 前必须校验对应 `mind_map_nodes` 节点存在。

## 存储与膨胀控制

- 不用硬性限制运行时内存作为主要方案。
- 通过按需加载、编辑器卸载、快照 hash 复用、快照保留数量、二进制资产外置来控制膨胀。
- 导图快照当前保留 12 份。
- Word 快照当前保留 16 份。
- 图片、附件、二进制内容后续必须进入资产表，不能以内联 base64 塞入导图或 Word JSON。

## 代码边界

- Electron main 进程拥有 MySQL、文件、更新、Chrome 端口和 IPC。
- React renderer 只拥有 UI 状态和编辑器挂载。
- Renderer 禁止直接访问 MySQL。
- `simple-mind-map` 的私有实例只能出现在 `simpleMindMapAdapter.ts`。
- `canvas-editor` 的私有实例只能出现在 `canvasEditorAdapter.ts`。
- 业务组件只能通过适配器暴露的稳定 handle 调编辑器能力。

## 修改准入

后续任何功能如果要改以下内容，必须先更新本契约，再改代码：

- 节点 ID 生成和归一化规则。
- 目录父子级生成规则。
- Word 文档绑定键。
- 导图或 Word 快照协议字段。
- MySQL 当前指针表和快照表职责。
- 编辑器适配层暴露的 handle 方法。
